MeiliSearch在windows上的安装&配置

MeiliSearch在windows上的安装&配置

1、环境说明

2、服务下载

在GitHub上可以下载最新的windows服务包，截至当前文章时间最新版本为v1.9.0;或在上面下载其他版本。
下载好后，在D盘下新建目录MeiliSearch,将exe包放在里面。

3、配置&启动

3.1、官网文档

在MeiliSearch官网可以找到本地环境安装的文档说明， 其中windows上安装的相关说明如下图,因为我们下载的是已经编译好的exe文件，所以看第三条，这里我本地已经安装了MinGW，所以后续启动没有问题，具体这些的区别可以参考Cygwin/MinGW/MSys/WSL区别与联系

3.2、环境变量

MeiliSearch有几个比较重要的变量，可以在执行时候命令行追加也可以在环境变量中维护，为方便维护记忆遂选择后者。这里可参考文章MeiliSearch V0.25使用教程，下文部分摘抄于此

数据库路径
  • 环境变量： MEILI_DB_PATH
  • 命令行： –db-path
  • 默认值： “data.ms/“
  • 期望值： 文件路径
  • 描述： 指定创建和检索数据库文件的位置

这里我在D:\MeiliSearch目录下新建了db目录，并在环境变量中进行配置

运行环境
  • 环境变量： MEILI_ENV
  • 命令行： –env
  • 默认值： development
  • 期望值： production / development
  • 描述： 配置实例的环境。值必须是production或development

这个变量我没有配置，本地启动的时候默认development，在启动日志中会打印

地址和端口
  • 环境变量： MEILI_HTTP_ADDR
  • 命令行： –http-addr
  • 默认值： “127.0.0.1:7700”
  • 期望值： 一个http地址和端口
  • 描述： 设置 Meilisearch 将使用的 HTTP 地址和端口

未配置，使用默认值

主秘钥
  • 环境变量： MEILI_MASTER_KEY
  • 命令行： –master-key
  • 默认值： None
  • 期望值： 字符串
  • 描述： 设置实例的主密钥，自动保护除 GET /health 之外的所有api。这意味着您将需要一个 API 来访问MeiliSearch，如 POST /search 和 GET /documents.

这里我在第一次启动时候未配置，后来发现在首次启动时候会自动生成，遂使用生成的进行配置

3.3、启动服务

在上文的路径下打开命令窗口，直接执行meilisearch-windows-amd64.exe,首次启动（未配置主密钥，按文档如果在启动前已配置应该不会自动生成）如下

将生成的密钥复制，按环境变量配置进行配置后重新打开新的命令窗口，再次执行exe文件

启动成功后可以在浏览器打开http://localhost:7700/ ,按提示输入主密钥即可进行使用了。
